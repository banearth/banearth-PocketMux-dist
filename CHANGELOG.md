# 更新日志

## 0.1.55
- **根治「串台」——聊天页/终端偶尔把别的 agent 的会话内容挂到当前 agent 头上**:病根是**客户端的持久缓存按 pane id(`%N`)做键**。rmux 的 pane id **不稳定**:daemon 每次重启会按 `agent_cwds.json` 顺序重建整支舰队(`restore_fleet`),只要 agent 有增减,**所有 pane id 整体漂移**——今天 `%2` 是 Rmux,重启后 `%2` 可能变成 Book。而客户端把「会话缓存」「已读状态」「我的发言历史」都按旧的 `%N` 存进 `localStorage`,于是重启后打开 Rmux,会先秒显示挂在 `%2` 旧键下的 **Book 会话**(串台);此前「终端串台」也是同一族根因。**修法两条**:①**所有按 agent 持久化的状态改用稳定键 = rmux 会话名**(`name`,唯一且跨重启不变)——会话缓存 `pmc:<name>`、已读 `pmSeen` 内层按 name、发言历史 `pmHist:<name>` 全部换掉;会话推送里只带 pane id,客户端维护一份 **id→name 映射**(每次舰队推送刷新、打开会话时也补一次),**映射缺失时宁可只进内存不落盘**,绝不用可疑的键写缓存。②**页面加载即清理旧遗留**:扫一遍 `localStorage`,凡是键以 pane id(`%` 开头)命名的会话/发言缓存、以及 `pmSeen` 里 `%` 开头的内层条目,**一律删除不迁移**(内容无法安全归属,宁可重拉也绝不冒串台风险);检测只认 `%` 前缀(tmux pane id 恒为 `%`+数字,而会话名绝不以 `%` 开头,零误伤),且**每次加载都跑**,任何来路的旧格式键都活不过一次打开。桥端**不改**(id→name 由现有舰队数据提供)。此改动**需重新构建桥**(chat.html 内嵌),无需重装安卓版。
- **修复「聊天页把模型偶发泄漏的『格式错误工具调用』渲染坏了」(两处)**:模型偶尔会把工具调用当成普通文本吐出来(原文里出现裸的 `<invoke name="Bash"><parameter …>…</invoke>` 标记),Claude Code 拒绝解析、注入一条「Your tool call was malformed and could not be parsed. Please retry.」的反馈让它重试。终端里显示正常,但**聊天页**渲染成两个错误:①那段原始标记被 marked 当**真 HTML 直接塞进 innerHTML**,浏览器把 `<invoke>`/`<parameter>` 当未知标签**吞掉**,气泡里只剩零散文字、不知所云;②注入的 malformed 反馈被当成**用户蓝气泡**,像是用户自己说的。现在:**(a) 通用转义**——给 marked 的 `html` 渲染器(块级+内联的裸 HTML 都走它)加转义,任何 `<div>`/`<img>`/`<invoke>` 等原始标签一律**按文本显示不再被吞**(代码块、行内代码、`![](/file?…)` 图片走各自的渲染路径,**不受影响照常工作**;顺带把 `<script>` 之类也中和了);**(b) 折叠泄漏的工具调用**——识别到 assistant 文本里的裸 `<invoke name=` 标记,整段折叠成一个 ⚠ `<details>` 块(风格同行内 diff 折叠):标题「⚠ 模型输出了格式错误的工具调用(已自动重试)」,展开是转义后的原文;**(c) 反馈变系统提示**——桥端按 Claude Code 的 `isMeta` 结构字段(malformed 反馈这类注入的 user-role 行都带 `isMeta:true`)给消息打 `meta` 标记(协议加法,默认不占字节),前端渲染成**居中灰色小字系统提示**,不再冒充用户蓝气泡。此改动**需重新构建桥**(chat.html 内嵌),无需重装安卓版。
- **修复「开 VPN 后,文件界面点『安装到手机』报 TypeError: Failed to fetch」**:开 VPN 时,已建立的 WS 长连接照常存活(界面一切正常),但**为下载 APK 而新建的 TCP 连接连不到桥**(桥日志证实请求根本没到达),于是 `fetch(/file?p=…)` 直接抛 `Failed to fetch`,安装失败。现在**安装不再只依赖 HTTP 直连**:直连一旦整段失败(网络层 TypeError),自动**降级走已存在的 WS 长连接分块拉取**——toast 提示「直连失败,改走备用通道…」,然后 512KB 一块循环拉完整个包、进度显示「备用通道 xx%」,拿到的字节照旧喂给原生分块安装管道调起系统安装器。桥端新增 `fetch_file` 请求走**后台通道**(不占交互 lane,拉大包时发消息照常秒回),路径安全校验**复用 /file 端点同一把锁**(canonicalize + 必须在某个 agent 目录/上传目录/临时目录之下,越权路径一律拒绝),单块 30s 无响应即超时;直连与备用都不通时报人话并附上双通道各自的失败详情。此改动**无需重装安卓版**(复用现有原生管道)。

## 0.1.54
- **修复「设置页滑不到底,流量卡片下面的内容够不着」**:设置面板在一个高度钉死、自身不可滚动的容器里(整页唯一能滚的是下面的 agent 列表),以前设置内容一屏装得下、没暴露;加了流量+Token 两张卡片后**超出一屏的部分被直接裁掉,手指怎么滑都到不了**(手机屏幕矮先撞上)。现在设置面板自己可滚动。教训:模拟器当时用 JS 程序化滚动验收(能越过裁剪截到图),而真手指的触摸滚动在那种容器上根本不生效——**滚动类验收一律用真触摸手势**。
- **新增「🧮 Token 统计」(设置页「📶 流量统计」下面,想对 AI 用量有更强感知)**:追踪自己的 token 花在哪。卡片展示**今日**(输出/输入/缓存读)、**今日按 agent Top5**(每个 agent 今日的输出/输入)、**最近 7 天**每天的输出/输入/缓存读,以及**30 天合计**。数据源是本机各 agent 的会话日志:Claude Code 的 `~/.claude/projects/**/*.jsonl` 里每条 assistant 消息的 `usage`(按 `requestId` 去重——一次请求会分成多行流式写入、带同一份 usage,不去重会重复计数),Codex rollout 的 `token_count` 事件(用累计值逐事件求差得每轮增量)。按**本地日**分桶(UTC 时间戳→本地日历日)。计算全在桥端后台线程做:按文件 mtime 缓存分日聚合,首次全量扫一次、之后每 ~60s 只重扫改动过的文件,**绝不在连接请求路径里扫大日志、也不碰 daemon**(纯读文件);WS 新增 `{type:"tokstats"}` 请求走后台通道返回预先算好的快照,首扫没完时显示「统计中…」。设置页打开时拉一次、停留期间每 10s 刷新(关掉即停)。
- **新增「📶 流量统计」(设置页,想对流量花在哪有更强感知)**:在 5G 下用手机远控,终端实时流/图片/预取到底谁吃流量,现在一眼可查。设置页「下载安卓版」下面多了一张卡片,展示**今日**与**本次会话**(自页面加载起)的 ↓下行/↑上行/合计,以及**今日分类明细**(终端实时流/会话内容/舰队与状态/图片与文件/页面与库/APK/上行)和**最近 7 天**每天的收发量。计量在应用层就地做:主 WebSocket 每条消息按类型(`termdata`→终端流、`conversation`→会话、`fleet`/`digest`→舰队状态、`upload`→图片上传……)累加真实 UTF-8 字节;HTTP 资源(图片、xterm.js/字体等库、APK 下载)用 `PerformanceObserver` 读真实传输字节。数据只存本机 `localStorage`、保留最近 30 天、超期自动清理,**不走网络**。注:是**应用层近似值**,不含 TCP/TLS 开销与通知服务的后台推送。
- **修复「刚打开终端、随手上下滑两下就闪两下」**:病根是**尺寸收敛永远收不拢**。前端每次打开终端会做几轮「校准适配」——字体就绪(`document.fonts.ready` + 0.5s 兜底)各重量一次、首帧关键帧到达后再收敛一两次——每轮都拿「本地算出的行数」和「pane 实际行数」比,不一致就**重新订阅**;而每次重订阅桥都回一张关键帧,前端处理关键帧要 `xterm.reset()`(**清屏**)再异步 `write`(**重画**),这一清一画之间就是一次**可见闪烁**。问题在于比较的行数**天生对不上**:前端请求的行数是「可视行数 + 80 行回滚余量」(可达 120),而桥把单个 pane 的行数**硬顶在 100**——申请 120、只给 100,`本地行数 !== pane行数` **永远成立**,于是每一次校准都触发重订阅→关键帧→清屏重画。实测刚打开终端 ~3 秒内会连闪 **5 次**,正好盖住用户最初的几次滑动(「滑两下闪两下」)。**修法**:行数只需**盖住可视区**即可(桥就顶 100,申请再多也是白申请,不该算「没适配好」),改成只有**列数变了**(真正的宽度/旋转/字号变化,会触发 TUI reflow)或 **pane 真的矮到盖不住可视区**(有露黑边风险)才重订阅——列数一直是对齐的,于是打开终端从 **5 次清屏重画降到 1 次**(那一次是必需的首帧上屏,不是闪)。复测:刚打开即滑 10 次、以及打开 30s 后滑 10 次,**零重订阅、零清屏、零几何抖动**。
- **顺手加一道手势保护**:钉底重滚(`termPin`)在**手指还按在屏上**(touchstart~touchend 之间)时**只重裁不重滚**——任何来自键盘/视口/定时兜底的钉底都不会在滑动中途把画面从手底下拽走。裁剪(保证下方黑边不可达)照常执行,只是不跟手指抢滚动位置。

## 0.1.53
- **手机上直接安装 agent 造出来的 APK(需重装安卓版 0.39)**:在「文件」界面点到任意 `.apk`(📦 图标),预览页给一个显眼的「安装到手机」按钮,点一下就把这个包**下载到手机并调起系统安装器**——于是 PC 上 agent 刚 build 出来的安卓包,能直接装到手边这台手机上验,不用再传来传去。机制上有个坑:App 的 WebView **没有下载监听**,`location.href` 指向附件在 App 内是死路,所以传输走的是 **fetch 拿字节 → JS 桥 → 原生**;大包(几十 MB)不能一次性塞进 base64 字符串(会撑爆桥),于是原生新增**分块接收**接口,边收边写磁盘、最后走 FileProvider 调安装意图,内存占用恒定。下载/传输**全程显示百分比**;桌面浏览器打开则退化为普通下载链接。**与「设置 → 下载安卓版」那条 App 自更新链路完全独立**,是给任意 APK 用的通用机制。⚠️ **原生改动:手机上要「设置 → 下载安卓版」重装一次,确认设置页显示 v0.39。**
- **根治「键盘一弹出,底部界面就堆叠/残影」(需重装安卓版 0.38)**:折腾了好几轮 CSS/JS 都压不住的顽疾,病根最终定位在**原生层**——App 的 WebView 被强制用**软件渲染层**(当年为治聊天列表的 GPU 残影加的),这种层把整个页面缓存成一张位图;键盘弹出/收起会**改变 WebView 尺寸**,而 resize 时**旧位图不被可靠丢弃**,旧帧像素就和新布局叠在一起(「堆叠」),直到某次全量重绘才消失。在模拟器上用改显示尺寸(和键盘走同一条 resize 路径)**稳定复现**了残影。0.37 先试了「resize 后重建渲染层」的保守修法,真机仍复发;0.38 **彻底移除软件渲染层、回到系统默认的硬件渲染**——resize 由 Chromium 合成器原生处理,这一类残影**整个消失**(WebView 上强制软件层本就是官方不支持的用法)。复测:复现操作 + 8 连击快速 resize **零残影**;当年软件层要治的聊天列表滚动残影也做了回归检查,未复现(若在个别真机上回潮,将用针对性 CSS 定点治,不再回退到软件层)。另外**设置页现在显示「安卓 App 版本」**,装没装上一眼可查。⚠️ **原生改动:手机上要「设置 → 下载安卓版」重装一次,并确认设置页显示 v0.38。**
- **已读状态跨设备同步**:手机上看过的对话,电脑上那个绿点也会跟着消失——**已读记录改由主机(桥)统一保存**(按 agent 会话名记,两台设备谁先看算谁的、只进不退),不再各存各的本地缓存。于是手机看完、电脑刷新,舰队列表上的「未读」标记就清了。
- **打包等宽中文字体**:终端字形改用**内置的等宽中文字体**渲染,不再看手机系统装了什么字体的脸色,中英文对齐、缺字方框都稳了。
- **修复打开 `/model` 等菜单后终端一片空白**:某些 agent 弹出选择菜单(如 `/model`)后终端看不到内容的问题已修。
- **修复上滑被弹回**:终端里向上翻历史不再被自动拽回底部。
- **终端下方空白区不再可滚动**:内容不足一屏时,下方空白区域不会再跟着乱滚。
- **发送后无响应会标注「未收到回应·重发」**:消息发出去迟迟没动静时,气泡会显式标注可点重发,不再干等。
- **发图片终于稳了(三个叠加的 bug 一起修)**:以前手机发图经常只发出文字、图丢了。根因有三层:① 客户端在图片**还没上传完**就把消息发出去了 → 图片路径是空的被丢掉(现在改成**等上传落地再自动发**);② 上传请求被排到了**后台预取队列**里,被一堆预取会话堵在后面;③ 更深的一层——桥每次都**内联重算整支舰队状态(18 个 agent 逐个抓屏),要十几秒,把整条连接都卡死**,回复/上传的确认全被饿死("发送中卡好几秒"就是这个)。现在舰队状态改为**后台算好缓存**、连接直接读缓存(**不再卡**),上传/回复/按键走独立优先通道,实测上传确认从十几秒降到**秒回**。
- **全局更跟手**:承上,桥的写出改成**独立优先队列**(交互帧插队、后台预取让路),不再被大块预取写堵住读取——聊天、终端、发送整体都更跟手。
- **修复「终端只有一张旧快照、没有实时内容」(僵尸流)**:实时流的转发器进程根本没起来——桥把自身 exe 路径(**反斜杠**)拼进 `pipe-pane` 命令,经 pane 的 shell 时反斜杠被当转义吃掉(`\t`→制表符、`\r`→回车),路径被打散、转发器自然起不来,偏偏 `pipe-pane` 还假装成功返回 0(所以只有打底快照、没有后续实时字节)。改成**正斜杠**路径后转发器正常连上,终端恢复逐字节实时刷新。
- **修复「打开终端整个缩放 / 下方一大片空白」**:根因两层。① pane 曾被每次抓屏的 `capture_terminal` 强制成 100 列 × 200 行,把订阅端设好的尺寸当场冲掉——已删掉那个强制 resize。② 更隐蔽:前端算「一格多高」时把行高**写死成 `字号 × 1.2`**,而 xterm 实际渲染的行高随字号在 **1.2~1.31 之间浮动**(实测 12px 是 14.4、14px 是 18.4),估出来的行数跟真实渲染对不上——可视区能放的行数一算错,底部就露黑边。改成**直接读 xterm 渲染引擎量出的真实格子尺寸**来算(等价于 xterm 的 FitAddon,只是没打包那个插件、直接读同一份数),列宽/行高都用量出来的真值,底部严丝合缝贴着输入框、无黑边、无横向溢出。
- **终端上下翻历史修回来了(我上一版把它改没了)**:上一版为了消掉黑边把 pane 压成「刚好一屏」,结果没有可回滚的历史、往上滑纹丝不动。现在 pane 高度改回**比可视区高一截**(可视行数 + 一段回滚余量),claude/codex 会把对话历史铺满这段高度——于是能往上滑看更早的内容、松手自动回到底部的输入框;满屏时最后一行(输入框+底栏)正好压在可视区底部,不多留空、也不切掉。
- **修复「点输入框、键盘弹出那一瞬下面按钮栏被压扁」**:键盘弹起时 WebView 可视区变矮,原来 Esc/方向键那一排、命令栏、输入栏会被 flex 一起挤扁一下再回弹。现在把这些底部栏都设成**不可压缩(`flex-shrink:0`)**,高度变化**只由终端区自己吸收**,按钮栏纹丝不动。
- **修复「在终端里点几下输入框、下方就冒出一大片黑」**:软键盘弹起/收起会改可视区高度,而现代安卓 WebView 的软键盘改的是 **visualViewport**(不是 layout viewport),`window.resize` 常常压根不触发 → 终端没机会重新钉底,滚动停在**露出 pane 尾部空行**的位置,那就是那片黑。**根治**:把 xterm 的可视网格**按内容高度裁掉尾部空行**(给 `.xterm` 定死 `height=内容高 + overflow:hidden`),于是 `#termScreen` 的**可滚动区就等于内容本身**——**任何滚动位置(包括键盘收起后停留的那个位置)都不可能再露出黑边**,是从结构上杜绝,而不是靠「等某个事件来重新钉底」的时序运气;另加 `visualViewport` / `ResizeObserver` + **每 0.6s 一次的定时兜底重算**三重保险(个别安卓 WebView 这三种事件都不可靠地报键盘开合,靠定时也能在半秒内自愈——比如键盘弹着时打 `/model`、收键盘后本会残留一小段留白,现在会自动补正)。实测:强制滚到底、模拟键盘开合(可视区 800→157→597)、以及把底对齐留白手动清零,终端下方黑边都在半秒内**回到 0**。
- **修复「刚打开终端那一下、字挤成一小团变形」**:xterm 第一次量「一个字符多大」时,若打包字体还没真正就绪(手机首次加载偶发——`document.fonts.load` 在安卓 WebView 上会**提前 resolve**),它会按后备字体量出一个**极小的格子**并缓存下来,字就被压成一小团糊在一起;字体随后到位它也不会自己重量。现在:① 量出的格子**小得离谱(宽<3 或高<8)就判为无效**、先用字号估算兜底(也顺带防止裁剪把界面剪没);② 终端一打开就挂上 `document.fonts.ready`(外加 0.5s 兜底)**强制 xterm 重量格子并重绘**——字体到位后自动恢复正常字号。(实测:把字体临时换成别的会立刻改变量出的格子,再调用重量逻辑能精确复原回打包字体的尺寸。)
- **修复「连接终端中…」占位字在手机上小得离谱、还赖在已加载的内容上面**:两处。① 这个「连接中」覆盖层本该**只在首次连接时显示、拿到第一帧就藏**;但字体重量/自适应触发的**复订阅**每次都把它又显示出来,于是它赖在已经加载好的终端内容上(你截图里那个)。现在**只有首次订阅才显示它**、且**收到任何一个字节就立刻藏**,不再赖着。② 占位字本身改用**显式 `font` 简写(字体族+字号+行高)、去掉 z-index、加 `text-size-adjust:100%`**、去掉表情符——部分手机的字体自动放大会放大正文却跳过这个绝对定位层、让它显得极小,这些改动尽量让它稳。桌面/模拟器(连 DPR 3.5 都试了)复现不出「变小」,只能验证桌面无回归 + 覆盖层现在确实**加载后就 `display:none`**、不再赖在内容上。
- **修复「AI 明明在思考却显示空闲」**:工作中的 spinner 行(`✶ 动词… (Ns · ↓ N tokens)`)排在待办列表+输入框上方,超出了原本只看「最后 6 行」的判定窗口,而唯一的 `esc to interrupt` 窄屏下又会被截断 → 误判空闲。现在**放宽到最后 12 行专门找 spinner**(省略号+括号 `… (`,或 `↓/↑` 配 tokens 的实时计数)。拿 18 个实景 pane 回归验证:只把「真正在跑的那个」从空闲纠正为运行中,其余 17 个空闲判定原样不动(零误伤)。
