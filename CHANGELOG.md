# 更新日志

## 0.1.49
- **打开会话快多了(增量解析 + 后台预热)**:以前每次点开 agent 会话,桥都要把整段会话日志(claude 最大 ~39MB、codex rollout 最大 ~135MB)**从头解析一遍**,动辄等十几秒。现在改成**只解析新写入的那几行**;而且桥会在**后台提前把每个 agent 的会话都解析好备着**(最近活跃的先备)—— 所以你点开任意一个(哪怕从通知点进、或刚看完别的 agent),解析早就做完了,基本**秒开**。实测某 38MB 会话:冷解析 15.3s → 增量 6.6ms。
- **图片不再每次重下(需重装一次安卓版 0.36)**:带大量图片的会话(比如逐帧看视频那种 agent),以前每次点开、每张图都重新下载、先是一片空白。现在图片**缓存在手机本地、留 7 天**,再点开秒显示。⚠️ 这条是**原生改动**,得在「设置 → 下载安卓版」**重装一次 APK(0.36)** 才生效(桥的自动更新不会替你换手机上的 App)。

## 0.1.48
- **手机上直接看 agent 每次改了啥(内联 diff)**:agent 每做一次 `Edit`/`Write`,回复气泡下面就多一个**默认折叠**的 diff 块(`📝 文件名 +N −M`),点开看具体改动。默认**统一单列**(手机友好、长行自动折行),标题右边 **`⇆ 双列`** 可切成 Rider 那种左右对照(记住选择)。同一个文件在一条回复里改多次会**聚合成一个块**(带 `×次数`,展开逐条看);**建了又删的临时文件不显示**(只留磁盘上仍存在的)。目前支持 claude 版 agent(codex 的补丁式编辑后续再加)。
- **(附带)安卓 APK 刷新到 0.35**:断线/首屏那页(App 自带的那份)的调试面板等修复也进包了 —— 想要的话在「设置 → 下载安卓版」重装一次即可。

## 0.1.46
- **诊断留痕：桥卡死/冻结现在能从日志一眼看出**：新增 `bridge.log` 心跳——桥每 60 秒写一行 `[hb]`（独立线程，异步运行时死锁也拦不住它）。规律的 `[hb] ok` = 存活（哪怕没人连）；`[hb] … WEDGED` = 异步运行时卡住；`[hb]` 直接中断 = 进程死亡。以前「桥不响应」只能看到一段日志空白、分不清是冻死还是没人用，现在带时间戳、一眼定性。另外控制台在 UI 落后丢日志行时（窗口隐藏被系统节流）会记 `UI live-view throttled: dropped N`，把界面卡顿的诱因也显式记下（此时桥本身不受影响）。**无功能改动，纯排障能力增强。**

## 0.1.44
- **修复「点了应用内更新就卡死、版本却纹丝不动」(重要)**：更新器三个 bug 叠加,点更新后卡在「主机重启中…等待自动重连」——① 更新器进程没脱离控制台的 kill-on-close 进程组,它一 `taskkill` 掉控制台,自己也被连带杀掉,一个文件都没拷就死了；② 用 `DETACHED_PROCESS` 起 PowerShell → 它没有控制台、脚本一行没跑就退出；③ `taskkill` 在严格模式下遇到「进程已不存在」会抛异常、直接中断更新(且每次必中)。已改为:更新器 `CREATE_BREAKAWAY_FROM_JOB` 挣脱进程组、`CREATE_NO_WINDOW`(而非 DETACHED)、并用 `Stop-Process -SilentlyContinue` 杀进程。⚠️ **本次必须手动更新一次**:发起更新用的是旧版本里那个坏更新器,所以从 ≤0.1.43 升到 0.1.44 没法靠应用内更新完成——请**下载 0.1.44 安装包覆盖安装**(配置在 `%APPDATA%\PocketMux`,重装不丢数据/token)。装上 0.1.44 之后,以后的更新才真正一键无感。

## 0.1.43
- **agent 生成的图片现在能显示了**：让 agent「随便生成张图看看」，它通常把图写在系统临时目录（`%TEMP%`）再刻意读出来给你——但之前渲染只放行**工程目录 / 上传目录**下的图，临时目录里的图会被安全校验挡掉：URL 发得出、手机去取的那一下被 403 拒掉 → 一片空白（既看不到图、也看不到 `![]()` 文字）。现在把系统临时目录也纳入放行。仍然安全：承接上版「**只渲染被刻意用看图工具（Read / view_image）看过的图**」，shell 顺带列出来的临时图依旧不显示，所以放开临时目录不会重新把无关图串进来。

## 0.1.42
- **修复上下文里偶尔串入无关图片**：渲染聊天时，之前会把 agent 用工具时「顺带列出来」的本地图片路径也当成图片画出来——比如一次宽泛的 `rg --files` 搜索把某个无关项目 assets 目录下的图（甚至别人的赞赏码/广告图）扫进了工具输出，就被渲染进当前会话，看着像「供应商串了别人的聊天」。改为**只渲染你真正分享的图**：你上传的图、assistant 正文里显式写的 `![]()`、以及 agent **刻意用看图工具（Read / view_image）看的那张图**；搜索 / shell 工具（rg / ls / grep）以及任何工具**输出**里冒出来的图片路径，一律不再渲染。判据是「这张图有没有被当成图片刻意处理过」，跟它在哪个目录无关。（补充：这些图本来就只是被查看器画出来、**从未**作为图片输入喂给模型。）

## 0.1.41
- **更新/重启不再打断正在跑的 agent**：rmux 内核（daemon）现在会从控制台的进程组里「挣脱」出来独立存活，所以更新桥/界面、或托盘「重启桥」时，**daemon 和它下面所有正在跑的 agent 都不受影响**，手机也只是瞬断重连。⚠️ 从旧版升到本版的**这一次**仍会打断一次（升级由旧版执行，挣脱能力要等本版起来才生效）；建议**在 agent 空闲时更新**，之后就彻底无感了
- **重启/掉线后自动复原整支 agent 队伍**：daemon 真的没了（关机、手动杀）时，下次启动会照记录把每个 agent 的会话按原工作目录重建、并**接回它原来的对话**（claude/codex resume）。配套：在手机上删除 agent，会**同步清掉**它的全部配置，不再残留「僵尸」
- **手机「连不上好几个小时、只有回电脑点一下托盘才活」彻底修复**：根因是桥的日志输出被控制台界面「反压」——界面缩在托盘被系统节流时，桥会卡在写日志上整个冻住。已把「排空日志」和「界面显示」**彻底解耦**，桥的存活不再依赖界面是否可见，人不在电脑边也不会再假死
- **文件浏览器正确渲染 Markdown**：打开 `.md` 现在按排版显示（标题/列表/表格/代码块），内嵌的本地图片也正常展示，不再是一坨原始源码
- **调试面板显示问题彻底修好**：把面板改成真正的顶层浮层（不再嵌在会被隐藏的容器里、也不再靠运行时搬家自救），任何页面、任何连接状态下点🐞都能正常弹出
- **内核（rmux）更新也能走自动更新**：更新器现在会比对内核是否变化——只有内核真的变了才重载 daemon，否则只换桥/界面（无感），内核更新不用再手动重装。⚠️ 本次更新器改动要等升到本版之后的**下一次**更新才生效（更新脚本由发起更新的旧版本写出）

## 0.1.40
- **自更新加固 + 留痕**：修好「点更新→进度转圈→最后弹『更新未完成(超时)』、版本却纹丝不动」的静默失败。更新脚本改为 `Stop` 严格模式 + try/catch，每一步（解压、逐个覆盖、校验 `version.txt`）都写进 `bridge.log`；覆盖前**轮询等旧程序解锁最多 15 秒再重试**，仍失败会明确记录 `COPY FAILED ...（文件被占用 / 无写权限 / 杀软拦截）`。轮询重试本身多半能直接治好「exe 没及时解锁」这类竞态。⚠️ 本次修复要等升到 0.1.40 之后的**下一次**更新才生效（更新脚本由发起更新的旧版本写出）
- **彻底修复聊天「乐观回显」重复**：你发出的消息本地先乐观显示一条，服务器数据回来后应替换掉它；旧的去重判据在长对话里（消息按 80 条滑动窗口传输）会失效，导致回显与真实消息并存、永久双份。改用会话的**绝对消息总数增长**判定回显已被确认（配合「最后一条用户消息内容匹配」兜底），任意长度对话都不再残留双份
- **聊天页显示 resume id（点按复制）**：续接中的会话，标题栏旁新增一枚 `📋 <id>` 小标签，点一下复制 `claude --resume <id>`（Codex 则 `codex resume <id>`），方便在独立控制台里接着同一会话
- **电脑浏览器拖拽文件→路径**：从资源管理器把文件拖进聊天区，其**原始绝对路径**自动填进输入框（带空格自动加引号）。浏览器出于安全不暴露本地路径，故由桥在当前 agent 的项目目录里按「文件名 + 大小」定位、返回真实路径；目录外的文件找不到会提示

## 0.1.39
- **修复更新日志在部分电脑上乱码**：更新面板里的「逐版本更新日志」是桥用 PowerShell 拉取的，之前会按该电脑的控制台代码页重新编码——UTF-8 代码页的机器正常，GBK/ANSI 代码页的机器就把中文变成乱码（同一版本因电脑而异）。改为直接输出服务器返回的原始 UTF-8 字节，任何代码页都不再乱。装上本版后，从下一版起更新说明即恢复正常显示

## 0.1.36
